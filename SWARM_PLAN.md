## Phase 3: λ-RLM core algorithm [PENDING]
- [ ] 3.1: Create apps/server/src/rlm/LambdaPlan.ts. Faithful TypeScript port of reference lambda_rlm.py _plan() method. Exports: TaskType enum (SUMMARIZATION|QA|TRANSLATION|CLASSIFICATION|EXTRACTION|ANALYSIS|GENERAL), ComposeOp enum (6 values), PipelineFlags type { useFilter: boolean }, LambdaPlan type { taskType, composeOp, pipeline, kStar, tauStar, depth, costEstimate, n }, COMPOSITION_TABLE, PLAN_TABLE, C_COMPOSE, C_IN constants (identical values to reference). plan(taskType, n, contextWindowChars?, accuracyTarget?, aLeaf?, aCompose?): LambdaPlan pure function implementing the exact reference formula: k*=ceil(sqrt(n*C_IN/c_compose)), d=ceil(log_k*(n/K)), accuracy constraint loop, tau*=min(K,floor(n/k*)), cost estimate. Defaults: contextWindowChars=100_000, accuracyTarget=0.80, aLeaf=0.95, aCompose=0.90. [MEDIUM]
- [ ] 3.2: Create apps/server/src/rlm/LamCodeExtractor.ts. Pure function extractLamCode(raw: string): string. Priority order: (1) content inside ```lambda...``` or ```lam...``` fence — strip fence markers and trim; (2) find first line matching /^@main\s*=/ in raw text, take that line and all subsequent lines until blank line or end; (3) raw string trimmed as fallback. No Effect imports. [SMALL]
- [ ] 3.3: Create apps/server/src/rlm/LambdaRlm.ts. Main λ-RLM class as an Effect service. Config type LambdaRlmConfig: { modelId: string; contextWindowChars: number; accuracyTarget: number; aLeaf: number; aCompose: number; maxDepth: number }. Reads LLM_MODEL (default 'minimax/minimax-m2.5:free') and RLM_MAX_DEPTH (default 3) from process.env. Implements the 5-phase algorithm from the reference as Effect.gen functions: Phase 1: context_0 stored as a string (no REPL socket needed — direct in-memory). Phase 2: one LLM call via OpenRouterClient using buildTaskDetectionProbe, parse digit → TaskType. Phase 3: plan(taskType, n, contextWindowChars, ...) → LambdaPlan — 0 LLM calls. Phase 4: log cost estimate via Effect.log. Phase 5: executeΦ recursive function — for lambda tasks (depth=0 from plan), effectiveDepth = max(plan.depth, config.maxDepth) drives self-correction iterations. Φ(context: string, depthRemaining: number): on leaf (depthRemaining==0 or len<=tauStar): call LLM via buildSolvePrompt or buildRetryPrompt → extractLamCode → runTask (lam checker as oracle) → return LlmCheckResult. On non-leaf: split(context, kStar) → Effect.all(chunks.map(Φ(..., d-1)), {concurrency: kStar}) → reduce (for lambda: selectBest by pass then by fewest errors). Main export: rlmEval = Effect.fn('rlmEval')(function*(task, refBits?)): Effect<LlmCheckResult, never, OpenRouterClient | FileSystem | Path>. LlmCheckResult extends CheckResult with { attempts: number; depth: number }. [LARGE]
